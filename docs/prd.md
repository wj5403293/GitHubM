# 需求文档

## 1. 应用概述

### 1.1 应用名称
GitHub 管理器

### 1.2 应用描述
一款通过 GitHub API 深度集成的管理器应用,支持 GitHub 令牌登录,提供响应式设计,适配手机端和 PC 端,集成 GitHub 网站的所有核心功能,支持完整的创建、删除、编辑等精细化操作。采用专业应用级美学设计,支持深色模式、浅色模式及跟随系统设置,通过右键上下文菜单提供便捷的操作体验。

## 2. 用户与使用场景

### 2.1 目标用户
- GitHub 用户(个人开发者、团队成员、项目管理者)
- 需要便捷管理 GitHub 资源的用户

### 2.2 核心使用场景
- 个人项目管理与维护
- 团队协作与代码审查
- 仓库监控与数据追踪
- 移动端快速处理 GitHub 事务
- 精细化文件与仓库管理操作

  - Star/取消 Star
  - Fork
  - Watch/取消 Watch
  - 删除仓库
  - 创建分支
  - 查看 Actions
  - 查看 Issues
  - 查看 Pull Requests
  - 部署设置

## 4. 业务规则与逻辑

### 4.1 登录与认证
- 用户首次访问时跳转至登录页
- 验证 GitHub 令牌有效性后允许访问
- 令牌保存至本地,用于后续 API 调用
- 令牌失效时自动跳转至登录页
- 支持多账号令牌同时保存,用户可随时切换

### 4.2 数据持久化
- 保存所有账号的 GitHub 令牌
- 保存用户收藏的仓库列表
- 保存用户自定义的筛选条件和排序偏好
- 保存浏览历史记录
- 保存当前使用的账号标识
- 保存用户选择的主题模式(深色/浅色/跟随系统)

### 4.3 响应式适配
- PC 端:采用多栏布局,充分利用屏幕空间
- 手机端:采用单栏布局,优化触控操作体验,采用精美的专业应用级美学设计
- 自动根据设备屏幕宽度切换布局

### 4.4 API 调用规则
- 所有功能通过 GitHub API 实现
- 使用用户提供的令牌进行身份验证
- 处理 API 限流情况,展示剩余请求次数

### 4.5 代码编码处理
- 获取文件内容时自动检测编码格式
- 优先尝试 UTF-8 解码
- 若出现乱码,依次尝试 GBK、GB2312、ISO-8859-1 等编码
- 正确显示中文及其他非 ASCII 字符
- 针对 Markdown 文件,在渲染前先进行编码检测和转换,确保内容正确显示

### 4.6 多账号切换
- 切换账号时自动使用对应令牌重新加载数据
- 不同账号的数据独立存储,互不影响
- 切换账号后保持在当前页面,仅刷新数据

### 4.7 批量上传规则
- 单次上传文件数量不超过 100 个
- 所有文件使用同一提交信息
- 上传过程中显示实时进度
- 若部分文件上传失败,显示失败文件列表及原因

### 4.8 网站部署规则
- 仅支持已启用 GitHub Pages 的仓库
- 部署源可选择分支(如 main、gh-pages)和目录(根目录或 /docs)
- 部署完成后自动生成访问地址
- 自定义域名需在仓库设置中配置 CNAME 记录

### 4.9 产物下载规则
- 支持单个文件直接下载
- 批量下载时自动打包为 ZIP 格式
- 下载前显示文件总大小

### 4.10 删除操作规则
- 所有删除操作均需用户二次确认
- 删除仓库时需输入仓库完整名称进行验证
- 删除文件夹时需显示将被删除的文件数量
- 删除操作执行后显示成功提示
- 删除失败时显示具体错误原因

### 4.11 创建操作规则
- 创建文件夹时自动创建 .gitkeep 占位文件(GitHub 不支持空文件夹)
- 创建文件时自动检测文件名合法性
- 创建操作需指定目标分支
- 所有创建操作需提供提交信息

### 4.12 批量操作规则
- 批量删除时显示操作进度
- 批量操作失败时显示失败项列表及原因
- 批量操作支持取消
- 批量操作完成后显示成功和失败的统计信息

### 4.13 右键上下文菜单规则
- 针对仓库的右键菜单在仓库列表项上触发
- 针对文件的右键菜单在文件树中的文件项上触发
- 针对文件夹的右键菜单在文件树中的文件夹项上触发
- 针对搜索结果中仓库的右键菜单在搜索结果仓库项上触发
- 菜单项根据当前对象类型和状态动态显示
- 菜单操作执行后自动关闭菜单

### 4.14 主题模式规则
- 支持三种主题模式:深色模式、浅色模式、跟随系统设置
- 跟随系统设置时,自动检测系统主题并应用
- 用户手动切换主题后,保存用户选择
- 主题切换时平滑过渡,无闪烁

### 4.15 文件图标显示规则
- 根据文件扩展名自动匹配对应图标
- 文件夹使用统一的文件夹图标
- 未识别类型的文件使用默认文件图标

### 4.16 图片预览规则
- 支持 JPG、PNG、GIF、SVG、WebP 等常见图片格式
- 点击图片文件时在预览窗口中显示
- 预览窗口支持缩放和关闭操作

### 4.17 代码编辑器全屏模式规则
- 全屏模式下隐藏其他界面元素,仅保留编辑器和必要控制按钮
- 提供明显的退出全屏按钮
- 全屏模式下保持所有编辑功能可用
- 退出全屏后恢复原始布局

### 4.18 文件 Raw 链接规则
- 复制 Raw 链接时自动生成文件的原始内容访问地址
- Raw 链接格式为:https://raw.githubusercontent.com/用户名/仓库名/分支名/文件路径
- 复制成功后显示提示信息

### 4.19 搜索结果仓库右键菜单规则
- 搜索结果中的仓库项具有专用右键菜单
- 菜单项针对搜索场景优化,提供快速访问和操作功能
- 菜单操作不影响搜索结果列表状态

## 5. 异常与边界情况

| 场景 | 处理方式 |
|------|----------|
| 令牌无效或过期 | 提示用户重新输入有效令牌 |
| API 请求失败 | 显示错误提示,支持重试 |
| API 限流 | 显示剩余请求次数,提示用户稍后再试 |
| 网络断开 | 提示网络异常,部分功能不可用 |
| 仓库不存在或无权限访问 | 提示相应错误信息 |
| 创建/修改操作失败 | 显示具体错误原因,引导用户修正 |
| 空数据状态 | 显示友好的空状态提示 |
| 文件编码无法识别 | 提示编码异常,建议使用 UTF-8 编码 |
| Markdown 文件渲染乱码 | 自动尝试多种编码格式解码,若仍失败则提示编码异常 |
| 代码编辑冲突 | 提示文件已被修改,需先拉取最新版本 |
| 数据导出失败 | 显示错误提示,支持重试 |
| 第三方集成配置错误 | 显示配置错误详情,引导用户修正 |
| 批量上传文件过多 | 提示超出数量限制,建议分批上传 |
| 批量上传部分失败 | 显示失败文件列表及失败原因,支持重新上传失败文件 |
| 仓库未启用 GitHub Pages | 提示需先在仓库设置中启用 GitHub Pages |
| 部署失败 | 显示部署失败原因,支持重新部署 |
| Release 无产物文件 | 显示提示信息,说明该版本无可下载产物 |
| 产物下载失败 | 显示错误提示,支持重试 |
| 删除仓库时名称输入错误 | 提示名称不匹配,禁止删除操作 |
| 删除受保护分支 | 提示该分支受保护,无法删除 |
| 创建重名文件/文件夹 | 提示名称已存在,要求修改名称 |
| 文件名包含非法字符 | 提示文件名不合法,列出非法字符 |
| 批量操作部分失败 | 显示成功和失败的统计信息及失败原因 |
| 移动文件到不存在的路径 | 提示目标路径不存在,要求修正 |
| 删除正在使用的分支 | 提示无法删除当前分支,需先切换分支 |
| 右键菜单在不支持的区域触发 | 不显示菜单或显示默认浏览器菜单 |
| 图片格式不支持预览 | 提示该格式暂不支持预览,提供下载选项 |
| 全屏模式下浏览器不支持 | 提示浏览器不支持全屏功能 |
| 系统主题检测失败 | 默认使用浅色模式 |
| 复制 Raw 链接失败 | 显示复制失败提示,支持重试 |
| 搜索结果为空 | 显示无结果提示,建议调整搜索条件 |
